# Dotfiles redesign: the plan

The grilling session is complete.
Every open question in the previous version of this document has been answered.
The reasoning lives in `docs/adr/`; the vocabulary lives in `CONTEXT.md`.
This file records only what is left to build, and the evidence that would be expensive to gather again.

Nothing here has been enacted yet.

## The actual problem

The stated problem was "manage dotfiles across Mac, Linux and Chromebook Crostini".
The real workload is one Mac used daily, plus a continuous stream of devcontainers.
"I copy the config manually across each" never meant copying dotfiles into running containers.
It meant copy-pasting the container environment itself into every new repo.

## Findings, verified, do not re-derive

- **chezmoi was never installed.** The repo adopted its naming convention and nothing else. Home files are ordinary files, not symlinks and not chezmoi-applied.
- **The repo is a 2023 snapshot.** Eight commits in three years. Live `~/.zshrc` and the repo's `dot_zshrc` have diverged in opposite directions, as have the two `~/.ssh/config`.
- **There are no secrets.** `~/.ssh` holds no private key at all; 1Password serves them over its agent socket. The only identifier in the tree is AgileBits' published Apple team ID. The repo is public, so this is now an invariant.
- **`alias ll='ls -al'` exists in seven files**, not the five previously counted. Three of the four sibling alias files contain nothing else.
- **Hard-won fixes do not propagate.** `actual-budget-transformer` puts `brew shellenv` in `/etc/zsh/zshenv` because `~/.zshrc` is read only by interactive shells. `homelab-kube` runs zsh without the fix and still has the bug.
- **`docker compose --build` does not defeat the layer cache.** Measured: a rebuilt `RUN` line reported `CACHED` and returned its original timestamp. Only `--no-cache` re-runs it.
- **Eager completions are expensive.** `uv generate-shell-completion zsh` costs 165ms per shell start; `gh completion -s zsh` costs 34ms. A `command -v` guard on an absent tool costs 0.018ms. Completion files on `fpath` are read on first Tab, or never.
- **`chezmoi status` exits 0 whether or not it reports drift.** Its first column is an uncaptured edit; its second is an unapplied change.
- **Dead code.** `homelab-fun/.devcontainer/install_1password_cli.sh` is referenced from nowhere. So is `raspi-custom-img/.bash_aliases`, whose only apparent reference points at a different file.

## Target state

```
~/.local/share/chezmoi/        the repo, on a durable target
  install.sh                   generated by `chezmoi generate install.sh`
  .chezmoiignore               contains install.sh
  .chezmoi.toml or config      mode = "file"  (the default)
  dot_zshrc                    the only shell file
  dot_config/starship.toml     includes the drift indicator
  private_dot_ssh/config       private_ on the directory yields ~/.ssh at 0700
  AGENTS.md
  dot_claude/symlink_CLAUDE.md -> ~/AGENTS.md
```

`~/dotfiles` on an ephemeral target, because that is where the platform clones it.

## Work, in dependency order

1. **Restructure this repo.** Delete `chromebook/` and `private_Library/`. Adopt `~/AGENTS.md`, `~/.config/starship.toml`, and the `~/.claude/CLAUDE.md` symlink. Reconcile `dot_zshrc` and `private_dot_ssh/config` with the live files they have drifted from, taking the live file as the winner where it is newer.
2. **Generate the installer.** `chezmoi generate install.sh > install.sh`, `chmod a+x`, `echo install.sh >> .chezmoiignore`.
3. **Write the drift indicator** into `starship.toml` as a custom module. It must test for empty output rather than an exit code, distinguish the two status columns, report unpublished and unfetched commits from cached refs, refresh those refs with a `git fetch` at most once a day behind a stamp file, skip ephemeral targets, and never remediate.
4. **Adopt on the Mac.** `chezmoi init --apply afrossard`. This is the first moment anything changes outside the repo.
5. **Set `dotfiles.repository`** to `afrossard/dotfiles` as a VS Code **user** setting. Every VS Code devcontainer and Codespace is then covered.
6. **The base-image repo.** It does not exist yet and this work does not block on it. It must install zsh, `chsh -s /bin/zsh`, put `brew shellenv` in `/etc/zsh/zshenv`, write tool completions into a site-functions directory on `fpath`, install the chezmoi binary, and stop writing `~/.zshrc` entirely.
7. **`actual-budget-transformer`.** Give the `claude` stage an entrypoint that runs `chezmoi init --apply` at start. Move its seven real aliases from `.zsh_aliases` into a file its `postCreateCommand.sh` drops at `~/.zshrc.d/`.
8. **The other three siblings.** Delete their alias files and the shell-config blocks in their Containerfiles and `postCreateCommand.sh`.
9. **Delete the dead code** found above.

## Deferred, with the mechanism known

- **Application-rewritten settings**, meaning `~/.claude/settings.json` and VS Code's `settings.json`. The mechanism is a chezmoi `modify_` script. See ADR-0001.
- **Sharing VS Code extension lists.** See ADR-0001.
- **Nix and home-manager.** A decision about tool installation, to be made in the base-image repo. If it is ever made there, revisit ADR-0003, because home-manager would then be nearly free.
- **A machine-provisioning repo** for the Chromebook's `sommelier` drop-ins and gtk settings. Not needed until the Chromebook is rebuilt.
